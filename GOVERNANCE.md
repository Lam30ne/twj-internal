# Document Governance Model

This repository contains internal Wonder Jam / Wonderly leadership materials.

## Document Types

### 1. Core Operating Docs
- Source of truth for how the business runs
- Must be kept current
- Examples: Operating Framework, Strategic Brief

### 2. Reviews & Audits
- Point-in-time analysis
- NOT source of truth
- Must include date context

### 3. Founder / Personal Briefs
- Highly sensitive
- Restricted audience

### 4. Team Communications
- Internal alignment
- Medium sensitivity

## Required Metadata (for future implementation)
Each document should include:
- Owner
- Status (Active / Superseded / Draft)
- Sensitivity (Internal / Restricted / Confidential)
- Effective Date
- Last Reviewed Date

## Key Rule
If a document is not marked ACTIVE, it should not be treated as a source of truth.

## Immediate Risk Note
This repository is currently public. If documents are confidential, hosting and access control must be addressed outside of this repo.
